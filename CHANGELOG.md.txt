# Changelog
All notable changes to Ruby Market Scanner are documented here.

###CHANGELOG.md — v1.6.0 (2026‑03‑13)
[1.6.0] – 2026‑03‑13

Added
- Weighted Composite Engine
Introduced a full agreement‑based weighting system for all indicators (MACD, slopes, VWAP, RSI, BB, ATR trend).
Each indicator now contributes a weighted bullish/bearish polarity, producing a smooth, expressive confidence score (e.g., 72.5% instead of binary 0/100).
- Composite Reason Narrative
Composite output now includes a detailed explanation of which indicators contributed to the bias and their weights (e.g., “MACD bullish (w3); Long‑term slope bullish (w3)”).
- Unified Composite Extractor (compute_composite)
Standardized access to composite bias, confidence, and reasoning across the entire GUI and scan pipeline.

Improved
- GUI Output Consistency
The Top‑5 section now displays only the weighted composite signals, removing the old “Top 5 Bullish Assets” block to avoid duplication and confusion.
Composite confidence in the GUI now matches the detailed snapshot exactly.
- Scan Pipeline Integration
Weighted composite computation is now executed during the scan loop, ensuring every asset has fully populated composite fields before GUI rendering.
- SignalBundle Enhancements
SignalBundle now stores weighted composite fields (composite_bias, composite_confidence, composite_reason, composite_confidence_note) as first‑class outputs.
- Architectural Cleanup
Moved polarity scoring and composite logic into the interpreter layer, keeping GUI and engine responsibilities cleanly separated.

Fixed
- Incorrect 100% Confidence in Top‑5 Table
Resolved an issue where the GUI displayed 100.0% for all assets due to reading pre‑composite fields.
Now correctly reflects weighted composite confidence.
- Duplicate Composite Header in GUI
Removed redundant header and ensured a single, clean “Composite Signals for Top 5” section.
- Import Errors
Fixed missing score_note_polarity reference by colocating polarity logic with the composite engine.


###[1.5.9]

###Upgraded

###New Features
Composite Signals for Top 5 Assets
Ruby now prints a composite signal summary for all Top 5 bullish assets, not just the strongest one.
This gives users a clearer, more comparative view of market conditions across leading candidates.
Aligned Composite Table
Composite signal output is now column‑aligned for improved readability:
- Symbols left‑aligned
- Composite bias left‑aligned
- Confidence values consistently formatted
This creates a clean, professional terminal table that’s easy to scan.
Color‑Coded Composite Bias
Composite rows now use color‑coding based on bias:
- Green → Bullish Bias
- Red → Bearish Bias
- Neutral → Neutral or mixed signals
This visual layer makes trend interpretation faster and more intuitive.

###Structural Improvements
Clarification Between Ranking Score & Composite Signal
Added a clear explanatory block:
- Ranking Score compares assets against each other
- Composite Signal measures indicator agreement within a single asset
This prevents confusion when a top‑ranked asset has a neutral or low‑confidence composite reading.
Cleaned Up Output Logic
- Removed duplicated on_scan_complete definitions
- Removed duplicated auto‑chart blocks
- Ensured composite block prints once, outside the Top‑5 loop
- Ensured indentation and flow are correct and stable

###User Experience Improvements
- Output is now more structured and readable
- Strongest asset still auto‑charts after scan completion
- Composite table appears immediately after the Top‑5 list
- Messaging is clearer, more consistent, and more professional

###Summary
This update significantly improves Ruby’s scan output clarity, visual polish, and interpretability.
Users now get:
- A ranked list
- A color‑coded composite table
- A clear explanation of how the systems differ
- A detailed snapshot of the strongest asset
