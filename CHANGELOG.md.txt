### Ruby v1.6.4 — Unified Cadence Engine & Correct Intraday Behavior
Ruby 1.6.4 delivers a major quality‑of‑life improvement to intraday analysis by unifying cadence handling across the entire application. SEARCH and SCAN now use the same cadence vocabulary, the same Coinbase granularity mapping, and the same historical window depth — producing consistent, accurate charts and indicator outputs across all timeframes.
This release fixes a subtle but impactful mismatch between GUI cadence values and backend provider mappings, ensuring that 1m/5m/15m/30m/60m all resolve to the correct Coinbase granularities. As a result, Ruby now displays properly shifting intraday structure for crypto assets across all cadences.

What’s New
• Unified cadence normalization
The GUI now normalizes cadence values ("1m", "5m", "15m", "30m", "60m") into a consistent internal format used across all providers.
• Expanded Coinbase granularity mapping
cadence_to_coinbase() now supports both "1m" and "1min" style inputs, eliminating the mismatch between SEARCH and SCAN.
• Correct intraday behavior for crypto
SEARCH now requests the correct Coinbase granularity for all cadences, fixing the issue where 60m searches defaulted to 5m data.
• Consistent candle windows across SEARCH and SCAN
Both paths now fetch equivalent historical depth, ensuring charts and indicators shift meaningfully between timeframes.

Bug Fixes
- Fixed a cadence mismatch where SEARCH used "60m" but the backend expected "60min", causing incorrect granularity selection.
- Corrected Coinbase granularity fallback behavior (no more silent defaults to 5m).
- Improved debug output to show true candle counts instead of dict key counts.
- Normalized cadence strings to prevent GUI → backend drift.

Impact
Ruby now produces:
- accurate intraday charts
- correct indicator calculations
- meaningful differences between 1m/5m/15m/60m
- consistent behavior between SEARCH and SCAN
This release dramatically improves Ruby’s reliability as a technical analysis tool, especially for crypto assets where cadence matters most.

### Ruby v1.6.2 — Yahoo Migration, Alpha Removal, and Zero‑Credential Crypto/Stock Scanning
Released: March 2026
Ruby 1.6.2 is a major architectural upgrade that fully removes all Alpha Vantage dependencies, introduces a modern Yahoo Finance data pipeline, and enables frictionless crypto scanning using Coinbase’s public market data — no API keys required. This release dramatically improves onboarding, stability, and long‑term maintainability.

New Features
• Full Yahoo Finance Integration (Stocks + ETFs)
Ruby now uses Yahoo Finance as the primary provider for all equities:
- Intraday OHLCV (1m, 5m, 15m, 30m, 60m)
- Daily OHLCV fallback
- Clean, normalized candle format
- Zero API keys required
- Faster, more stable, and more reliable than Alpha Vantage
• Zero‑Credential Crypto Scanning (Coinbase Public API)
Crypto scanning now works out‑of‑the‑box:
- No API keys needed
- No onboarding friction
- No authentication errors
- Clean intraday candles from Coinbase public endpoints
- Perfectly normalized to match Yahoo’s structure
This dramatically improves first‑run experience for new users.

Improvements
• Complete Removal of Alpha Vantage
All Alpha‑era components have been fully removed:
- DataFetcher
- alpha_client
- Alpha normalization functions
- Alpha routing logic
- Alpha GUI references
- Alpha installer fields
Ruby is now 100% Alpha‑free.
• Unified Candle Normalization Layer
All providers now feed into a single, consistent candle format:
{
  "timestamp": ...,
  "open": ...,
  "high": ...,
  "low": ...,
  "close": ...,
  "volume": ...

This powers a cleaner, more stable indicator pipeline.
• Robust Candle Filtering
Added safeguards to remove malformed Yahoo candles:
- Filters out None OHLC values
- Removes zero‑value candles
- Sorts by timestamp
- Median‑based spike removal for Coinbase quirks
This prevents RSI/MACD crashes and ensures indicator stability.
• GUI Routing Updated to Match Scan Pipeline
The “Run Ruby Once” button now uses the same provider logic as the full scan:
- Yahoo intraday → daily fallback
- Coinbase for crypto
- No Alpha references
- No undefined variables (fetcher removed)
• Cleaner Logging & Debug Output
- Clear provider routing messages
- Graceful handling of unsupported Yahoo tickers
- Debug prints for spike removal
- Cleaner scan summaries

Bug Fixes
- Fixed ModuleNotFoundError: data_fetcher by removing all Alpha imports
- Fixed NameError: fetcher in GUI snapshot mode
- Fixed RSI crash caused by NoneType candles
- Fixed VWAP divide‑by‑zero warnings
- Fixed inconsistent symbol normalization for crypto
- Fixed GUI freeze when Yahoo intraday failed
- Fixed BRK.B, WBA, PARA, ATVI, HEXO, WISH handling (unsupported tickers now skipped cleanly)

Result
Ruby 1.6.2 is faster, cleaner, and dramatically easier to onboard:
- No API keys required for stocks
- No API keys required for crypto
- No Alpha Vantage throttling
- No credential setup friction
- Unified provider architecture
- Stable indicators across all assets
This is the most polished and user‑friendly version of Ruby to date.



###CHANGELOG.md — v1.6.0 (2026‑03‑13)
[1.6.0] – 2026‑03‑13

Added
- Weighted Composite Engine
Introduced a full agreement‑based weighting system for all indicators (MACD, slopes, VWAP, RSI, BB, ATR trend).
Each indicator now contributes a weighted bullish/bearish polarity, producing a smooth, expressive confidence score (e.g., 72.5% instead of binary 0/100).
- Composite Reason Narrative
Composite output now includes a detailed explanation of which indicators contributed to the bias and their weights (e.g., “MACD bullish (w3); Long‑term slope bullish (w3)”).
- Unified Composite Extractor (compute_composite)
Standardized access to composite bias, confidence, and reasoning across the entire GUI and scan pipeline.

Improved
- GUI Output Consistency
The Top‑5 section now displays only the weighted composite signals, removing the old “Top 5 Bullish Assets” block to avoid duplication and confusion.
Composite confidence in the GUI now matches the detailed snapshot exactly.
- Scan Pipeline Integration
Weighted composite computation is now executed during the scan loop, ensuring every asset has fully populated composite fields before GUI rendering.
- SignalBundle Enhancements
SignalBundle now stores weighted composite fields (composite_bias, composite_confidence, composite_reason, composite_confidence_note) as first‑class outputs.
- Architectural Cleanup
Moved polarity scoring and composite logic into the interpreter layer, keeping GUI and engine responsibilities cleanly separated.

Fixed
- Incorrect 100% Confidence in Top‑5 Table
Resolved an issue where the GUI displayed 100.0% for all assets due to reading pre‑composite fields.
Now correctly reflects weighted composite confidence.
- Duplicate Composite Header in GUI
Removed redundant header and ensured a single, clean “Composite Signals for Top 5” section.
- Import Errors
Fixed missing score_note_polarity reference by colocating polarity logic with the composite engine.
